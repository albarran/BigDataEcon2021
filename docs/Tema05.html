Limpieza y “doma” de datos (“data wrangling”)

  • Un análisis de datos adecuado requiere (mucho) tiempo previo de trabajo “sucio”, pero clave para obtener información útil en la toma de decisiones

  • dplyr (de tidyverse) ofrece herramientas de uso común, basadas en una gramática de manipulación de datos

    • Rendimiento rápido para operaciones en memoria

    • Similar e inspirado por declaraciones de consulta SQL

Los cinco verbos

  • La mayoría de operaciones en una tabla de datos pueden realizarse combinando cinco “verbos”:

1.- select(): selecciona columnas (variables)

2.- filter(): selecciona filas (observaciones)

3.-mutate(): añade nuevas columnas

  • también rename(): cambiar el nombre de una columna

Los cinco verbos (cont.)

4.- arrange(): re-ordena filas

5.- summarize(): colapsa un data frame a una fila;

  • Todos tienen como primer argumento un data frame, los siguientes describen qué hacer (con columnas o filas) y devuelven otro data frame

  • Es fácil encadenar pasos con el operador de tuberías (%>%)

  • Se pueden usar con group_by() para operar por grupo

Seleccionar variables y filtrar filas

  • select(): nombres o de posiciones de columnas, separados por comas
select(presidential, name, party)
select(presidential, 1:2, 4)
  • filter(): conserva filas en las que la condición lógica es verdadera
filter(presidential, party == "Republican")
  • Combinar filter() y select(): ej., presidentes demócratas desde 1973
select(filter(presidential, start > 1973 & party == "Democratic"), name)
  • La operación filter() se puede anidar dentro de la operación select(), porque cada verbo toma y devuelve un data frame
  • Estas largas expresiones se vuelven muy difíciles de leer

Anidamiento y el operador de tubería %>%

  • datos %>% filter(condition) equivale a filter(datos, condition)

  • El anidamiento es fácil:
    - Tomar presidential y pasarlo a filtrar (produce un nuevo data frame);
    - Tomar este resultado y pasarlo a seleccionar.

presidential %>%
  filter(start > 1973 & party == "Democratic") %>%
  select(name)

  • Atajo de teclado: Cmd / Ctrl + Mays + M

  • Se puede aplicar a cualquier función 

10 %>% log()               # = log(10)
presidential %>% head(2)   # = head(presidential, 2)
lm(data = mtcars, mpg ~ hp) %>%  summary()

Crear nuevas variables y organizar filas

  • mutate(): crear (o redefinir) variables según una fórmula a partir de otras

  • rename(): cambiar el nombre de una variable

mypresidents <- presidential %>%                 # evitar "machacar" una fuente original
                  mutate(duracion = end - start) 
presidential %>% mutate(sigloXXI = start > 2000,
                        duracion = end - start)  # crear varias separadas por comas

presidential %>% rename(nombre = name)
  • arrange(): ordena las filas todas las columnas de un data frame, en orden ascendente (por defecto) o descendente con desc()
mypresidents %>% arrange(desc(duracion))           
mypresidents %>% arrange(desc(duracion), start) # primero ordena por duración 
                                                # en casos de empate, por start

Resumir todo el conjunto de datos

  • summarize() contrae (colapsa) muchos valores de un data frame en una sola fila resumen

  • Tenemos que especificar cómo queremos reducir una columna entera de datos en un solo valor.

library("lubridate")
mypresidents %>%
  summarize(
    N = n(),                         # n() cuenta número total de filas
    first_year = min(year(start)),   # year() extrae el año de una fecha
    last_year = max(year(end)),
    num_dems = sum(party == "Democratic"),
    media_duracion = mean(duracion)
    )

Resumir para cada grupo

  • summarize() suele usarse en conjunción con group_by(): cambia el alcance para que no actúe sobre todo el data frame sino en grupos individuales

  • Ej., ¿cuál es la duración media de los demócratas y de los republicanos?

  • En lugar de hacerlo para cada partido por separado, especificamos que las filas deben ser agrupadas por los valores de esa variable

mypresidents %>% 
  group_by(party) %>%          # "marca" dos grupos en los datos 
  summarize(N = n(), 
            media_duracion = mean(duracion))

  • Nuevo data frame con distinto nivel de observación (fila): una fila para cada valor de la variable por la que se agrupa (ej., de presidentes a partidos)

  • ungroup() elimina la agrupación: volvemos a operar en datos desagrupados

Seleccionar muchas variables

library("nycflights13")           # incluye flights:  19 variables
select(flights, year:arr_time)    # desde variable "year" hasta "arr_time"
select(flights, -(year:day))      # todas menos "year, month, day"
  • Funciones a utilizar dentro de select():
    • starts_with("abc"): nombres que comienzan con “abc”.
    • ends_with("xyz"): nombres que acaban con “xyz”.
    • contains("ijk"): nombres que contienen “ijk”.
    • num_range("x", 1:3): para x1, x2 y x3.
    • matches(): nombres que coinciden con una expresión regular
presidential %>% select(party, everything())   # everything(): útil para reordenar

Manejo de valores ausentes NA

  • Como son “contagiosos”, una opción es eliminar observaciones con NA, PERO

    • en una variable, con argumentos como na.rm=TRUE o filter() y is.na() o drop_na(x)

    • o en todo el conjunto de datos con drop_na():

data <- tibble(x1 = c(1:4, NA, 6:7, NA), x2 = c(NA, 12:14, NA, 16:18) )

data %>% summarize(num = n(), meanNA = mean(x1), mean = mean(x1, na.rm = TRUE))

data %>% filter(!is.na(x1)) %>% 
  summarize(num = n(), meanNA = mean(x1), mean   = mean(x1, na.rm = TRUE))
data %>% drop_na(x1) %>% 
  summarize(num = n(), meanNA = mean(x1), mean   = mean(x1, na.rm = TRUE))

data %>%  drop_na() %>% 
  summarize(num = n(), meanNA = mean(x1), mean   = mean(x1, na.rm = TRUE))

Algunos verbos adicionales

  • slice(): extrae filas por posición

  • slice_sample(): extrae filas aleatoriamente

  • distinct(): extrae sólo las filas únicas (una o varias variables)
  • count(): cuenta los valores únicos de una o más variables
flights %>% count(tailnum) 
flights %>% group_by(tailnum) %>% summarize(n = n())
flights %>% count(tailnum, sort = TRUE)

  • count() también puede realizar recuentos “ponderados”

flights %>% count(tailnum, wt = dep_delay)   # suma de retrasos acumulados 

flights %>% group_by(tailnum) %>% summarize(n = sum(dep_delay))

Funciones útiles para crear variables

  • Operadores aritméticos: +, -, *, /, ^, %/% (división entera), %% (resto)

    • combinados con otros: x - mean(x), y - sum(y)

  • Logartimos, log(), retardos, lag(), adelantos, lead()

    • diferencia móvil, x - lag(x), ¿cambian el valor? x! = lag (x)

  • Agregados acumulativos y móviles: cumsum(1:10), cumprod(1:10), cummin(1:10), cummax(1:10), cummean(1:10)

  • Comparaciones lógicas: <, <=, >, >=, !=

  • Ordenamiento: min_rank(), row_number() y otras de dplyr::ranking 

y <- c (1, 2, 2, NA, 3, 4)
min_rank(y)               
min_rank(desc(y))
row_number(y)

Funciones útiles para crear variables (cont.)

  • if_else(): ejecución condicional (también case_when())
flights %>% mutate(retraso = if_else(dep_delay > 0, "retraso", "adelanto"))   
flights %>% mutate(retraso = if_else(dep_delay > 0, "retraso",             # encadenados
                                        if_else(dep_delay <0, "adelanto", "en hora")))
  • Nota: retraso debería convertirse a factor
  • Discretizar variables: cut_interval(), cut_number(), cut_width()
flights %>% mutate(retraso = cut_width(dep_delay,2))   # retraso es factor

  • parse_number(), parse_factor(), etc. son alternativas de dplyr a as.number(), as.factor() en lugar de factor

  • Nota: dplyr tiene muchas funciones equivalentes a otras de R base; bind_cols() y bind_rows() equivalen a cbind() y rbind()

Funciones de resumen útiles

  • Medidas de centralidad y de dispersión: mean(x), median(x), sd(x), IQR(x), mad(x)

  • Medidas de rango: min(x), quantile(x, 0.25), max(x)

  • Medidas de posición: first(x), nth(x, 2), last(x).

    • similar a x[1], x[2] y x[length(x)]

  • Conteos:

    • n(): observaciones totales (tamaño del grupo)

    • sum(!is.na(x)): observaciones no ausentes

    • n_distinct(x): filas distintas en x
