Introducción

  • R Markdown proporciona un marco unificado para el análisis de datos reproducible: combina código, resultados y comentarios

  • Útil como cuaderno de trabajo o para mostrar a quienes se centran en las conclusiones para tomar decisiones

  • Integra paquetes de R (rmarkdown y knitr) y herramientas externas para producir documentos html, PDF, Word, presentaciones, etc.

  • Recursos de de Ayuda

    • “Chuleta”: Help > Cheatsheets > R Markdown Cheat Sheet,

    • Guía de Referencia: Help > Cheatsheets > R Markdown Reference Guide.

    • Libro online sobre R Markdown.

Crear un documento de R Markdown

  • A partir de una plantilla en RStudio con o File > New File > R Markdown: se elige formato de salida (HTML, por defecto)

    • si rmarkdown no está instalado y cargado, RStudio pedirá instalarlo

  • Se compila o renderiza con (“tejer”) a la opción por defecto (html) o desplegar para elegir otros formatos

    • en el botón contiguo (engranaje) se pueden cambiar algunas opciones
  • Para crear PDFs, es necesario tener instalado una distribución de LaTeX
install.packages("tinytex")
tinytex::install_tinytex()
  • Practica a renderizar el documento de la plantilla en los tres formatos.

Formato de texto con Markdown

  • Los archivos .Rmd están escritos en Markdown: un conjunto ligero de convenciones para el formato de archivos de texto sin formato.

Formato de texto

*cursiva*  o _cursiva_
**negrita**   __negrita__
`código`
superíndice^2^ y subíndice~2~
~~tachado~~

Encabezamientos

# Encabezado de 1er. nivel
## Encabezado de 2º nivel

Listas

*   Item (elemento) de la lista (no numeradas)
*   Item 2
    - Sub-Item 2a (*, + y - son alternativas)
        + sub-sub-item
    - Sub-Item 2b

1.  Item 1 de la lista numerada
1.  Item 2. Los números se incrementan automáticamente.

1)  Otra forma de crear listas numeradas.
2)  También, se puede poner explícitamente el número.

Editor Visual de R Markdown en RStudio

  • En documentos .Rmd clic en el botón o elegir la opción con el botón de engranaje

  • En el modo visual, la barra de herramientas tiene accesos a todos los elementos de formato de texto y otros

    • También existen atajos de teclado (para RStudio, en general)

  • Se puede configurar la corrección ortográfica en Tools > Global Options > Spelling

    • agregar/seleccionar el diccionario de Español
    • marcar corrección en tiempo real

Fragmentos de código (“code chunks”)

  • Código en medio del texto: p.e., `r 2+3` muestra 5

    • con el atajo Ctrl + I, con icono (visual) , o escribiendo entre ` `

  • Fragmento de código (varias líneas)

    • en modo visual, icono y/o el desplegable

    • atajo de teclado Ctrl + Alt + I

    • escribiendo manualmente los delimitadores de fragmento

                    ``` {r}

                    ```

  • Útil para mostrar resultados en el documento o solo el código (ilustración)

Opciones básicas para fragmentos de código

\(\Rightarrow\) Estando en modo visual, en el icono de engranaje del fragmento, o bien separadas por comas en {r}

  • El fragmento puede tener un nombre para referencias

  • Mostar o no avisos, errores, mensajes al ejecutar el código

    • en el editor visual, marcar la opción para mostrar o no
    • establecer con = para warning, error o message el valor TRUE o FALSE

  • echo = TRUE para mostrar el código (y echo = FALSE para no hacerlo)

  • eval = TRUE para evaluar el código: los resultados de ejecutar el código están disponibles y pueden mostrarse (o no con eval = FALSE)

  • La lista completa de opciones aquí

    • (este enlace también aparece en el engranaje en visual)

Otras opciones para fragmentos de código

  • results = hide: no muestra los resultados

    • también otras opciones de cómo mostrarlos

  • fig.show: cómo mostrar la figura

    • una opción es no mostrarla: fig.show = hide

  • include = FALSE: no se incluye ni el código ni su resultado, pero se evalua

  • fig.width y fig.height (o fig.dim): fijar las dimensiones de una figura

  • out.width y out.height: dimensiones de una figura en el documento de salida (diferentes de lo anterior, que son las reales de la figura)

  • fig.align: mostrar la figura centrada o alineada a derecha o izquierda

Otras opciones para fragmentos de código (cont.)

  • fig.cap: leyenda de la figura

  • Combinando opciones para poner dos figuras una al lado de otra 

# ```{r, echo=TRUE, fig.show="hold", out.width="50%"}
plot(cars)                  # en la izquierda
plot(cars, type = "l")      # en la derecha

Opciones globales para fragmentos de código

  • Si no se da un valor a las opciones en un fragmento, se utilizan el valor por defecto global en el documento.

  • El comando opts_chunk$set() cambia esas opciones globales en un documento, por ejemplo,

    • knitr::opts_chunk$set(echo=FALSE, fig.width=6, fig.height=6)

  • Suele incluirse en un fragmento al principio del documento

    • podéis verlo en el documento de la plantilla, con nombre setup

    • se suelen cargar también las bibliotecas necesarias

    • notad el include = FALSE (para no mostar, pero sí evaluar)

    • también es recomendable omitir mensajes y warnings (al cargar algunas bibliotecas)

Ejecución de código en .Rmd

  • Al renderizar un documento .Rmd, se crea un nuevo espacio de trabajo y se ejecuta el código allí.

    • ese espacio de trabajo es distinto del que vemos en RStudio: diferentes objetos, incluyendo bibliotecas

  • Es importante conocer qué códigos evaluamos y los objetos disponibles en cada parte del documento

  • Si ejecutamos código del documento .Rmd (no renderizando), éste pasa a la consola y sí forma parte del espacio de trabajo de la sesión actual de R.

  • Se puede ejecutar línea a línea, un fragmento completo con o todos los anteriores con

  • Útil para probar el código (antes de renderizar), pero debemos asegurarnos de que la sesión incluye sólo código de fragmentos del documento

Mejorar la salida de tablas

  • Biblioteca printr: mejora la salida de (algunas) tablas en HTML

    • incompatible con df_print: paged

  • Biblioteca pander (avanzado): la función pandoc.table() convierto objetos de R a tablas en código de Markdown, con muchas opciones

  • Biblioteca xtable: el comando xtable() (y otros) presentar objeto de R de forma elegante en html y LaTeX

  • Biblioteca kableExtra (extiende la funcionalidad de knitr::kable(): ayuda.

  • Tanto xtable como kableExtra muestran “data frame”/“tibbles” como tablas

  • broom::tidy() (de tidyverse) convierte mucho objetos de R (en particular, salidas de comandos) en “tibbles”
