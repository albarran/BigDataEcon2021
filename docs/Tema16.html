Clustering

  • Para K dado, se divide el conjunto de datos en K grupos distintos \(\small C_1,\dots,C_K\) (no solapados)

  • Cada observaci√≥n pertenece a un grupo

  • La variaci√≥n dentro de un cluster (within-cluster sum of squares o wss) es una medida de cu√°nto difieren las observaciones de un cluster entre s√≠

\[\scriptsize W(C_{k}) = \frac{1}{|n_{k}|}\sum_{i,i^{\prime} \in C_{k}} \sum_{j=1}^{p}(x_{ij} - x_{i^{\prime}j})^{2}\]

  • La partici√≥n √≥ptima minimiza la suma total de la variaci√≥n within (total wss) \(\small \mathrm{min}_{C_{1},\ \dots,\ C_{K}} \bigg \{\sum_{k=1}^{K} W(C_{k}) \bigg \}\) \(\Rightarrow\) INVIABLE (\(\small K^n\) particiones)

Algoritmo de K-Means

  • Se puede encontrar un m√≠nimo local

  1. Conjetura inicial: asignar cada observaci√≥n a un grupo (aleatoriamente)

  2. Iterar hasta las asignaciones dejen de cambiar:

    a.- Calcular el centroide \(\small m_k\) de cada grupo(vector de medias de las observaciones del grupo \(\small k\))

    b.- Asignar cada observaci√≥n al grupo cuyo centroide es el m√°s cercano (distancia L2).

Algoritmo de K-Means: diferentes m√≠nimos

  • Este algoritmo garantiza que la funci√≥n objetivo disminuye en cada paso

  • El resultado (√≥ptimo local) depende de la asignaci√≥n aleatoria inicial

  • Se DEBE ejecutar el algoritmo con varias asignaciones iniciales y seleccionar la soluci√≥n con menor valor de la funci√≥n objetivo
