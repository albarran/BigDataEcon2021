Automatización de tareas

  • Los ordenadores son buenos para tareas repetidas, PERO NO para saber qué hacer o para interpretar resultados (el por qué)
  • “Si un bloque de código se repite más de dos veces, escribe una función”
datos <- tibble(a=1:10, b=21:30, c=51:60)

datos$a <- (datos$a - min(datos$a)) / (max(datos$a)-min(datos$a))
datos$b <- (datos$b - min(datos$b)) / (max(datos$b)-min(datos$b))
datos$c <- (datos$c - min(datos$c)) / (max(datos$c)-min(datos$c))
  • A diferencia de copiar-pegar, el código se modifica en un solo lugar
reescalar <- function(x) {
  ( x - min(x) ) / ( max(x) -  min(x) )
}

Definir una función

nombre_de_función <- function(arg1, arg2 = "valor") {      # nombre descriptivo
...
<código de R>
...
return(x)
}

  • Lista de argumentos de entrada:

    • obligatorio: arg1
    • opcional: arg2 (con valor por defecto “valor”)
    • argumento “especial” ...
    • información sobre los argumentos: formals()

  • Resultado: por defecto, la última línea u objeto imprimido o explícitamente con return()

Ejemplos de función

reescalar1 <- function(x) {
  num <- (x - min(x))
  num / ( max(x) -  min(x) )
}

\(\hspace{2cm}\)

reescalar2 <- function(x) {
  num <- (x - min(x))
  sol <- num / ( max(x) -  min(x) )
  return(sol)
}
reescalar3 <- function(x) {
  num <- (x - min(x))
  sol <- num / ( max(x) -  min(x) )   # RECORDAD: una asignación NO imprime nada
}
  • Las funciones deben ejecutarse antes de usarse: pasan a formar parte del entorno global de R
  • Se suelen incluir varias funciones en un archivo (=biblioteca) y se “incluyen/ejecutan” con source()

Ámbito o Alcance de las variables (“Scoping”)

  • La función tiene su propio “entorno”

  • Los objetos definidos en una función (incluidos los argumentos) tienen un ámbito local

    • NO se recuerdan fuera de ella.

    • PERO es preferible evitar un mismo nombre en el entorno global 

datos <- 1:5
reescalar2(datos)
sol                 # no se encuentra el objeto 'sol'

  • La función busca objetos (datos, variables, etc.) en el entorno global R solo si no los encuentra localmente (definidos o pasados como argumentos)

    • es preferible minimizar esta forma de proceder

Ejecución condicional (if)

  • Ejecuta un código según se cumpla una condición lógica
if (condición) {
  # código ecuando es cierta (TRUE)
} else {
  # código cuando es falsa (FALSE)
}

\(\hspace{2cm}\)

  • También se pueden encadenar múltiples condiciones
if (condición1) {
  # haz esto
} else if (condición alternativa) {
  # haz otra cosa
} else {
  # hacer en el caso residual
}

  • Un uso habitual: comprobar los argumentos de una función

  • Con stop() se detiene y devuelve un mensaje de error

  • Diferencia con if_else(): devuelve un objeto según la condición

Bucles / Iteraciones

  • Dos formas de evitar duplicidad:

    • funciones identifican patrones repetidos de código

    • iteraciones ayudan a repetir la misma operación con múltiples entradas

print(1)       # repetimos `print()` con tres entradas
print(2)
print(3)
  • Con un bucle for, una variable x itera sobre los valores de una secuencia
for (x in 1:3) {
  print(x)
}

\(\hspace{5cm}\)

x <- 1
print(x)
x <- 2
print(x)
x <- 3
print(x)

Elementos en la estructura de un bucle

output <- vector("double", 4)    # 1. resultado
for (i in 1:4) {                 # 2. secuencia
  output[[i]] <- mean(iris[[i]]) # 3. cuerpo
}
output                    # vector de medias

\(\hspace{1.5cm}\)

output <- list()    
for (i in seq_along(iris)) {
  output[[i]] <- mean(iris[[i]])
}
output    # lista o usar unlist()
  • Primero, reservar espacio para el resultado: un objeto existente o nuevo con vector() o list(), especificando tipo y longitud (no necesario)

  • La secuencia itera sobre un índice: ej. 1:20 o con seq_along(x)

  • También se puede iterar sobre elementos de cualquier vector: for (i in x) donde x es c("a", "b", "c" ) o c(1,15, 100, 200, 500)

  • Accedemos a los elementos con x[[i]]: tanto para listas y data frame como vectores (enfatiza acceso a UN elemento del vector)

Bucle while

  • Solo tiene dos componentes: se ejecuta el cuerpo mientras se cumpla una condición
while (condición) {
  # cuerpo
}
  • Se usa cuando la longitud de la secuencia desconocida: no se sabe cuántas veces se debe ejecutar un código
  • En ocasiones for y while son equivalentes
for (i in seq_along(x)) {
  # cuerpo
}

\(\hspace{2cm}\)

i <- 1
while (i <= length(x)) {
  # cuerpo
  i <- i + 1 
}

Las funciones map(): operaciones vectorizadas

  • Programación funcional y filosofía MapReduce como alternativa a los bucles: un problema complejo se descompone en piezas independientes
  1. Tomar una lista como entrada
  2. Aplicar a cada elemento una función (solución para una pieza simple)
  3. Devolver un nuevo objeto de la misma longitud (y nombres), con la clase dada por el sufijo de map()
map() lista map_lgl() vector lógico map_dbl() vector doble
map_dfr() data frame map_int() vector entero map_chr() vector de caracteres
  • Similares a lapply() y familia de R base

Las funciones map() (cont.)

mis_coches <- c("corolla", "civic")             # 1. Entrada/secuencia: vector, lista, df,...
show_cars <- function(mod = "civic", n = 6)  {  # 2. Función/cuerpo
  mpg %>% filter(model == mod) %>% head(n)
}
mis_coches %>% map(show_cars)                   # 3. Resultado, como lista
mis_coches %>% map_dfr(show_cars)               #               como df
  • También se pueden usar funciones incorporadas en R y usar ... para que el mapeo pase argumentos adicionales a la función
# `map()` es más rápido que `for()`

ptm <- proc.time()
iris %>% select(1:4) %>% 
  map_dbl(mean, na.rm=TRUE)
proc.time()-ptm

\(\hspace{2cm}\)

ptm <- proc.time()
output <- vector("double", 4)    
for (i in 1:4) {                 
  output[[i]] <- mean(iris[[i]])    
}
output                    
proc.time()-ptm

Iterando sobre grupos de un data frame

  • group_split() divide los datos en grupos definidos por un factor

    • group_by() solo “marca” (no modifica “físicamente” el objeto)

  • Así se puede usar map() para iterar en subgrupos de datos

  • Permite hacer más que summarize(): solo una línea por grupo

mtcars %>%
  group_split(cyl) %>%  # = base::split(.$cyl)
  map(head, n = 2)   
mtcars %>%
  group_split(cyl) %>%            
  map_dfr(head, n = 2)
  • También se podría usar group_by() más group_modify()

Otros usos de map()

  • Dentro de map() se pueden usar funciones “anónimas” definidas implícitamente con una fórmula, en lugar de con function()
models <- mtcars %>% 
  group_split(cyl) %>%              
  map(~lm(mpg ~ wt, data = .x))
  • Notad .x (o .) que indica “el objeto que pasa la tubería”: útil cuando lo que se pasa no es el primer argumento de la función
  • También se puede acceder a elementos de la lista de entrada por nombre o por índice de posición
models %>%  map(summary) %>% map_dbl("r.squared")

x <- list(list(1, 2, 3), list(4, 5, 6), list(7, 8, 9))
x %>% map_dbl(2)

Múltiples argumentos: map2() y pmap()

  • map2() permite pasar dos vectores a una función
mu    <- list(5, 10, -3)
sigma <- list(1, 5, 10)
map2(mu, sigma, rnorm, n = 5) %>% glimpse()

\(\hspace{1cm}\)

  • pmap() toma múltiples argumentos (pasados como lista con nombres o data frame para determinar su dimensión)
n <- list(1, 3, 5)
args1 <- list(n, mu, sigma)
args1 %>% pmap(rnorm) %>% glimpse()

\(\hspace{1cm}\)

Otras funciones relacionadas (purrr)

  • walk(), walk2() y pwalk() para funciones que realizan una acción en lugar de dar un resualtado (ej., print(), ggsave())
list(1, "a", 3)  %>% walk(print)
  • keep() y discard() para funciones que devuelven TRUE/FALSE
iris %>% keep(is.factor)

  • safely(), possibly(), quietly() para “capturar” errores

  • reduce() para funciones que reducen dos listas a una (uniones, intersecciones)

list(1:30, seq(5,40,4), 25:45) %>% reduce(intersect)
